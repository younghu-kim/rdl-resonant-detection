### 🚨 [이전 분석] 통일표 방법론 결함 — 켤레합 누락 가설 (부분적 기각)

#### 발견 경위

C-278 (ζ(s), ρ=-0.437)과 C-270 (ζ(s), ρ=-0.800)의 같은 함수에 대한 상관값이
2배 가까이 차이나는 원인을 추적한 결과, **영점 합 방법론의 근본적 차이**를 발견.

#### 코드 비교 결과

| 항목 | C-278 (통일표 방법론) | C-270 (안정성 검증) |
|------|----------------------|---------------------|
| 같은편 영점합 | ±300 이웃만 | **전체** 1517개 |
| 켤레 영점합 | **없음** ❌ | **있음** ✅ |
| Gamma 보정 | S1에서 차감 → A_L | S1에 가산 → A_Λ |
| 결과 ρ(gap_min) | -0.437 | -0.800 (A_L), -0.900 (A_Λ) |

**핵심**: C-278의 A_L 계산에서 켤레 영점합 `Σ 1/(γ₀ + γₖ)`이 누락.
이것은 수학적으로 **불완전한 로랑 계수**. primitive L-함수의 영점 전개에서
S₁(γₙ) = Σ_{k≠n} 1/(γₙ-γₖ) + Σ_k 1/(γₙ+γₖ) 이 올바른 형태.

#### 수치적 검증 (직접 수행)

C-278 데이터(337행) 재분석:
- 전체: ρ(A_L, gap_min) = -0.437
- 중앙 60% trim: ρ(A_L, gap_min) = -0.506
- t>400만: ρ(A_L, gap_min) = -0.562
- **어떤 처리로도 C-270의 -0.80에 근접 불가** → trim/t-범위 문제가 아님

C-278 t-bin 분석:
| bin | t̄ | ρ(A_bare,gm) | ρ(A_L,gm) |
|-----|-----|-------------|-----------|
| 1 | 128 | -0.28 | -0.15 |
| 2 | 283 | -0.59 | -0.38 |
| 3 | 415 | -0.60 | -0.59 |
| 4 | 538 | -0.48 | -0.54 |

→ 모든 빈에서 |ρ| < 0.60. C-270 bin-1 (t̄=626)의 -0.96과 비교 불가.

#### B_smooth 분석 (부수 발견)

- ρ(B_smooth, gap_min) = 0.004 (p=0.95) — **무상관 확인** ✅
- ρ(A_bare, B_smooth) = -0.50 (Pearson) — **음의 상관** → σ(A_L) < σ(A_bare)
- 감쇠 공식 ρ(A_L) ≈ ρ(A_bare)×σ(A_bare)/σ(A_L): 예측 -0.517 vs 실제 -0.437 (편차 0.08)
  → 감쇠 공식은 근사적으로만 성립. A_bare와 B_smooth가 독립이 아니기 때문.

#### 파급 영향

**통일표 11개 항목 모두 C-278 방법론 (켤레합 누락) 사용** → 전면 재측정 필요 가능.

| 시나리오 | 결과 | Paper 4 영향 |
|----------|------|-------------|
| 재측정 후 ρ ≈ -0.80 | damping 0.38 소멸, GUE에 근접 | **강화** (더 강한 증거) |
| 재측정 후 ρ ≈ -0.45 유지 | 켤레합이 ζ(s)에만 영향 | 현행 유지 |
| d≥2에서 효과 다름 | GL(d) 종속 damping | 표 갱신 |

### 구체적 실험 사양

**스크립트**: `scripts/zeta_conj_fix_c282.py` (C-278 기반 수정)

**수정 사항** (C-278 → C-282):

1. `compute_A()` 함수에 켤레 영점합 추가 (C-270 lines 200-233의 로직을 따를 것):
   ```python
   # 기존 (C-278): 같은편만, ±300 이웃
   S1_bare = Σ 1/(γ₀ - γₖ)  # k ∈ [idx-300, idx+300]
   
   # 수정 (C-282): 같은편 + 켤레, 전체 영점 (C-270과 동일)
   S1_same = Σ 1/(γ₀ - γₖ)   # k ≠ idx, 전체
   S1_conj = Σ 1/(γ₀ + γₖ)   # k = 전체 (자기 포함)
   S1_total = S1_same + S1_conj
   H1_same = Σ 1/(γ₀ - γₖ)²
   H1_conj = Σ 1/(γ₀ + γₖ)²
   H1_total = H1_same + H1_conj
   
   A_L = S1_total² + 2*H1_total          # primitive (Gamma 미포함)
   A_Λ = (S1_total - im_Γ)² + 2*(H1_total + re_Γ')  # completed
   ```

2. `N_MAX = 300` 제거 → 전체 영점 사용

3. T=2000 (C-270과 동일 범위)

4. 60% trim 적용 (C-270과 동일)

5. 출력에 A_bare_old (C-278식, 비교용), A_L_new (수정), A_Λ_new (수정) 모두 포함

**모델**: **sonnet** (C-270 코드에서 compute_A 로직 복사 + C-278 출력 포맷 유지. 수학적 판단 불필요, 코드 조합 작업.)
