**모델**: opus

**왜**: GL(2)에서 두 타원곡선(11a1, 37a1)은 모두 **weight 2**이다. 수학적 보편성을 주장하려면 **weight가 다른** GL(2) L-함수가 필수. 라마누잔 Δ 함수는:

1. **Weight 12** (vs weight 2): Γ-factor 구조가 다름. σ-유일성 패턴에 영향 가능.
2. **Level 1** (vs level 11, 37): 가장 단순한 conductor. bad prime 없음.
3. **수학사적 중요성**: Ramanujan τ(n) — 가장 유명한 modular form. 논문 임팩트 극대화.
4. **임계선 σ=6** (vs σ=1): Re(s) = k/2 = 6. 완전히 다른 위치.
5. **ε=+1, rank 0**: 11a1과 동일 부호이지만, weight와 conductor가 다름.

"weight 2에서의 3/4 패턴이 weight 12에서도 재현되면" → **GL(2) 프레임워크가 weight에 무관** → 진정한 보편성.

논문: "tested across weight 2 (elliptic curves) and weight 12 (Ramanujan Δ), levels 1, 11, 37"

### 설계 핵심

#### 대상: 라마누잔 Δ 함수 (Discriminant modular form)

- **Level N = 1** (SL₂(ℤ) 위의 cusp form)
- **Weight k = 12** (Δ(z) = q∏(1-qⁿ)²⁴)
- **Root number ε = +1**
- **임계선**: Re(s) = k/2 = **6**
- **함수 방정식**: s ↔ 12-s (대칭축 s=6)
- LMFDB: https://www.lmfdb.org/ModularForm/GL2/Q/holomorphic/1/12/a/a/
- **L-함수**: https://www.lmfdb.org/L/2/1/1.1/r0/0/0 (degree 2, level 1)

#### 수학적 구조

1. **L-함수**: L(Δ, s) = Σ τ(n) n⁻ˢ (Re(s) > 13/2)
   - τ(n) = Ramanujan tau function
   - τ(1)=1, τ(2)=-24, τ(3)=252, τ(4)=-1472, τ(5)=4830, ...
   - τ(n) multiplicative: τ(mn)=τ(m)τ(n) for gcd(m,n)=1
   - τ(p²)=τ(p)²-p¹¹ (Hecke 재귀)
   - **bad prime 없음** (level 1)
   - |τ(p)| ≤ 2p^{11/2} (Deligne 정리 — Ramanujan 추측 증명)

2. **완비 L-함수**: 
   Λ(Δ, s) = (2π)⁻ˢ · Γ(s) · L(Δ, s)
   
   ⚠️ **정규화 주의**: 두 가지 관례가 있음:
   - **대수적 정규화**: s의 중심 = k/2 = 6, Γ(s) 사용
   - **해석적 정규화**: s의 중심 = 1/2, Γ(s+11/2) 사용 (s → s+11/2 shift)
   
   **이 실험에서는 대수적 정규화 사용** (타원곡선과 통일):
   - Λ(Δ, s) = (2π)⁻ˢ Γ(s) L(Δ, s)
   - **함수 방정식**: Λ(Δ, 12-s) = **+1** · Λ(Δ, s)
   - **임계선**: Re(s) = 6

3. **ξ-다발 접속**: L = Λ'/Λ (타원곡선과 동일 구조)
4. **곡률**: κ(σ,t) = |L(σ+it)|²

5. **검증 항목** (4성질):
   - σ-유일성: σ=6이 특별한지? (weight 12의 Γ(s)가 다르게 행동할 수 있음)
   - 모노드로미: 영점 주위 ±π 양자화
   - κ 집중: near(σ=6)/far(σ≠6) >> 1
   - 블라인드 예측: κ 피크로 영점 위치 예측

#### 구현 — 타원곡선 스크립트 기반

**Step 1: τ(n) 계수 계산**
- 방법: q-전개 Δ(q) = q∏(1-qⁿ)²⁴에서 직접 계산 (가장 안전)
- Hecke 재귀로 교차검증: τ(p²)=τ(p)²-p¹¹
- LMFDB 참조값으로 교차검증 (n=1..20)
- 알려진 값: τ(2)=-24, τ(3)=252, τ(5)=4830, τ(7)=-16744, τ(11)=534612, τ(13)=-577738

**Step 2: AFE 구현** — 타원곡선 코드에서 수정
- Λ(Δ,s) = (2π)⁻ˢ Γ(s) L(Δ,s)
- AFE: 타원곡선과 동일 구조, N=1, ε=+1
- ⚠️ **τ(n)의 크기**: |τ(n)| ~ n^{11/2}로 매우 큼 → n⁻ˢ와 상쇄 (s≈6에서 n⁻⁶ · n^{5.5} = n⁻⁰·⁵)
- 수렴: AFE가 필수 (직접합 조건부 수렴만)

**Step 3: 함수 방정식 검증**
- Λ(Δ, 12-s) = Λ(Δ, s) 확인 (4점 이상)
- L(Δ, 6) 중심값 계산 → LMFDB 교차검증

**Step 4: 영점 탐색**
- t ∈ [0.5, 30] (σ=6 임계선 위)
- 영점: s = 6 + iγ, Λ(6+iγ) = 0
- LMFDB 교차검증: 첫 영점 γ₁ ≈ 9.2224 (확인 필요)
- Λ(6+it)는 실수 (ε=+1) → Re(Λ) 부호 변화로 탐색

**Step 5: 4성질 검증** — 타원곡선 패턴 답습

**Step 6: 결과 저장** → `results/ramanujan_delta_46.txt`

### 주의 (⚠️)

1. **τ(n) 크기**: τ(n) ~ n^{11/2}로 거대. mpmath 정밀도 80자리 유지 필수. overflow 주의.
2. **임계선 σ=6**: 모든 σ 관련 코드에서 하드코딩 확인. σ_crit = k/2 = 6.
3. **Γ(s) vs Γ(s+11/2)**: 정규화 관례 혼동 주의. 대수적 정규화 사용 시 Γ(s) 그대로.
4. **AFE 수렴**: N=1이므로 xₙ = 2πn/√1 = 2πn. 지수 감쇄가 빠름 → N_MAX=50으로 충분할 수 있음. 하지만 안전하게 100 유지.
5. **LMFDB 영점**: 대수적 정규화 vs 해석적 정규화의 영점 위치 차이 주의. LMFDB는 해석적 정규화(중심 1/2)를 사용할 수 있음 → γ_LMFDB + 11/2 = γ_우리 (shift 확인).
6. **11a1/37a1 스크립트 재활용**: `scripts/elliptic_curve_gl2_44.py`를 기반으로 수정.
7. **bundle_utils.py 수정 금지**: 별도 스크립트 내 구현.
8. **κ near/far 원시값 기록**: #44/#45 검토자 피드백 반영.
9. **σ-유일성**: weight 12의 Γ(s) 진동이 weight 2와 다를 수 있음. FAIL이 예상되지만, PASS이면 매우 흥미로운 발견.
