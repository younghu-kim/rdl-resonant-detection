### 다음 작업: **곡률장 수 분산(Number Variance) — RMT 연결 최종 검증**

**모델**: sonnet

**왜**:

27개 결과로 프레임워크의 수치적 검증이 포화 상태다. 그러나 **기존 랜덤 행렬 이론(RMT) 문헌과의 정량적 연결**이 아직 불완전하다. 결과 #20(GUE/Poisson 대조)에서 ρ(b)=-0.008은 정성적 확인이었으나, RMT의 핵심 예측인 **대수적 수 분산**(logarithmic number variance)은 미검증이다.

이 실험은:
1. **논문의 최종 피스** — 다발 기하학 ↔ RMT 정량 연결
2. **새로운 결과 유형** — 기존 27개는 개별 영점/불변량. 이것은 **통계적 집합 성질**
3. **심사자 설득력** — "bundle framework reproduces GUE number variance" 한 줄이 심사에 결정적
4. **정보 이득 보장** — GUE 확인이면 강한 양성, 이탈이면 새 물리 발견

**구체 설계**:

1. **플라켓 영점 계수 in 슬라이딩 윈도우**:
   - 영역: [0.3, 0.7] × [50, 600] (약 250개 ζ 영점 포함)
   - 격자: Nσ=32, Nt=2000 (충분한 해상도)
   - 슬라이딩 윈도우 폭 Δt: ⟨N⟩ = 5, 10, 20, 50에 해당하는 4종
   - 각 윈도우에서 N_plaq = (1/2π) Σ plaquette flux
   - 윈도우를 0.5Δt씩 이동하며 통계 수집

2. **수 분산 계산**:
   ```
   Var(N; L) = ⟨(N_plaq - ⟨N_plaq⟩)²⟩
   ```

3. **이론 비교**:
   - **GUE 예측**: Var(N; L) ≈ (2/π²)(log(2πL) + 1 + γ_E - π²/8) ≈ (2/π²) log L
   - **Poisson 대조**: Var(N; L) = L (선형)
   - **실제 ζ 영점**: 대수적 GUE 패턴이 알려진 결과 (Odlyzko, Berry)

4. **추가 검증 — 직접 영점 계수와 비교**:
   - 같은 윈도우에서 zetazero() 직접 계수로 Var 계산
   - 플라켓 기반 Var와 직접 기반 Var의 일치 확인 → 방법론 검증

5. **디리클레 확장 (선택)**:
   - χ₅의 영점으로도 동일 분석 → 보편성 확인

6. **계산량 추산**:
   - 32×2000 = 64000 격자점, t∈[50,600], dps=80
   - 영점 수: ~250, 윈도우 수: ~100-500 (크기별)
   - 예상: **20-40분**

7. **출력**: `results/number_variance_rmt.txt` + `results/number_variance_figdata.csv`

**주의**:
- 윈도우 끝부분(t<70, t>580) 경계 효과 → 양쪽 20%는 통계에서 제외
- dps=80 필수 (t~600에서 ξ 연산 안정성)
- 플라켓 격자가 충분히 조밀해야 영점 분리 — Nt=2000이면 간격 0.275, 평균 gap~1.1이므로 안전
- Number variance의 L은 **unfolded 좌표**에서의 간격 (평균 밀도로 정규화)

**성공 기준**:
- **완전 양성**: 4종 윈도우 모두 GUE 곡선 ±20% 이내 + Poisson과 >3σ 차이
- **부분 양성**: 일부 윈도우에서 GUE 일치, 나머지 경계 효과
- **음성**: Poisson 패턴 → 방법론 문제 (거의 불가능 — 영점은 GUE 통계 따름이 확립)

**논문 가치**: "The topological charge distribution reproduces GUE logarithmic number variance" — 다발 기하학과 랜덤 행렬 이론의 정량적 교량. 이후 논문 완성 단계 전환 가능.

### 알려진 함정 (누적)

- 모노드로미: eps 차분 금지 → 폐곡선 적분(radius=0.5, 64단계)

- κ 측정: 영점 위 직접 측정 금지 → δ=0.03~0.05 오프셋
- findroot: 단일 시작점만, 튜플 전달 금지
- numpy 2.0: np.trapezoid (trapz 아님)
- **bundle_utils.py h=10^{-20}**: dps ≥ 100 또는 해석적 공식 사용.
- **중간점 Hadamard 상쇄**: m=(γₙ+γₙ₊₁)/2에서 두 최근접 영점의 기여 = 0.
- **ξ(s) underflow**: log(ξ)' 해석적 공식으로 영구 해결.
- **mpmath.diff(xi, s)**: t>300에서 실패. 사용 금지.
- **Dirichlet κ 계산**: bare L'/L 사용 금지. Λ'/Λ 해석적 공식 필수.
- **윤곽 적분 시 영점 회피**: t₂는 반드시 영점 사이 중간점.
- **홀수 디리클레 지표 영점 탐색**: Re(L(1/2+it))=0 ≠ L=0. |L| 최소화 필수.
- **짝수 디리클레 지표**: a=0, Γ(s/2) 사용. L(1/2+it) ≠ 실수 (Λ만 실수). Re(L)=0 단독 불충분.
- **영점 탐색 보편적 방법**: |L(1/2+it)| 최소화가 Re(L)=0보다 엄밀히 우월 (A⊂B). 패리티 무관.
- **F₂ 면적분 정의**: ∂_σ Im(L)은 대칭 상쇄 → 격자 플라켓(arg ξ 와동 밀도) 사용. Stokes 등가.
- **σ-국소화 표현**: "RH의 기하학적 동치" 금지 → "RH와 일관적인 기하학적 증거" 수준으로. FWHM=0 → "FWHM < δσ".
- **다항식 log(0) 회피**: 격자점이 영점을 정확히 지나지 않도록 범위 조절.
- **Number variance unfolding**: L은 unfolded 좌표(평균 밀도로 정규화). raw t-간격 사용 금지.
