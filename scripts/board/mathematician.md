### Devil's Advocate: 세 가지 이슈 분리

**이슈 1: 모노드로미 bug (파트 C) — 코드 버그, 수학 실패 아님**

t=191.98까지 mono/π=2.000 (129개 연속 완벽), t=193.11부터 갑자기 mono/π=0.000 (7개).
**Λ(s) underflow** 원인:
- `completed_L(s) = (q/π)^{s/2} · Γ((s+a)/2) · L(s,χ)`
- t≈193에서 `|Γ((σ+1)/2 + it/2)| ~ e^{-πt/4}` → |Γ| ~ 10^{-66}
- dps=80 환경에서 Λ 절대값이 skip 임계값 근접 → arg 불안정/skip
- **해결**: `arg(Λ) = Im[log(Λ)]` 해석적 계산 (각 항 개별, underflow 없음)

**이슈 2: κ 비율 37× (파트 B) — 방법론 차이 + conductor 효과**

**현재** (buggy):
```python
val = completed_L(s, ci)  # ← Λ underflow
curr_arg = float(mpmath.arg(val))
```

**수정**:
```python
# arg(Λ) = Im[log(Λ)] (underflow 없는 log-space 계산)
q = mpmath.mpf(ci['q'])
a = mpmath.mpf(ci['a'])
log_prefactor = (s / 2) * mpmath.log(q / mpmath.pi)
log_gamma = mpmath.loggamma((s + a) / 2)
L_val = mpmath.dirichlet(s, ci['chi'])
if abs(L_val) < mpmath.mpf(10)**(-mpmath.mp.dps + 15):
    continue  # L 자체가 0이면 skip
log_L = mpmath.log(L_val)
log_Lambda = log_prefactor + log_gamma + log_L
curr_arg = float(mpmath.im(log_Lambda))
```

**실행**:
1. 함수 수정
2. 파트 C만 재실행 (136개 영점 하드코딩 또는 기존 파트 A 결과 재활용)

4. **진단 포함**: t=191.98과 t=193.11에서 기존 방식 vs log 방식 비교 출력

**주의**:
- ★ `mpmath.log(L_val)` 시 L_val 음수/0 방어
