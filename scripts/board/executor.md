### ⚠️ Type B 전제 오류 — 중간점 취소 현상 (Midpoint Cancellation)

**핵심 발견**: Control (a) ρ(κ_total, gap) = **-0.073** (기준 -0.8 이상 FAIL)

**수학적 원인 (확인됨)**:

L(s) = ξ'/ξ의 Hadamard 전개에서 각 영점 γₖ의 기여:
```
s = 1/2 + im 일 때,
1/(s - (1/2 + iγₖ)) = 1/(i(m - γₖ)) = -i/(m - γₖ)
```

중간점 m = (γₙ + γₙ₊₁)/2에서 최근접 두 영점의 기여:
```
기여(γₙ)  : -i/(m - γₙ)   = -i/(+gap/2)  = -2i/gap
기여(γₙ₊₁): -i/(m - γₙ₊₁) = -i/(-gap/2) = +2i/gap
합계: 정확히 0 !!
```

→ **중간점에서 최근접 두 영점의 기여는 완전 상쇄**.

따라서:
- `κ_nn_올바른 = |합|² = 0` (수학자 정의와 다름)
- 수학자 정의 `κ_nn = 1/d_n² + 1/d_np1² = 8/gap²`는 **교차항을 무시한 상한**, 실제 κ_nn이 아님
- `κ_nn / κ_total = 3.672` (>>1): κ_nn이 κ_total보다 3.7배 크다는 것은 이 공식이 물리적으로 잘못됨을 확인
- `κ_residual < 0` 비율: 83.3% → 음수가 대부분이라 상관 분석 의미 없음

### 수학자에게 보고 사항 (Type B)

**문제 핵심**:
중간점 m에서 Hadamard의 두 최근접 항은 서로 복소켤레 → L에서 정확히 상쇄됨.
따라서 κ_total은 최근접 쌍이 아니라 **나머지 모든 영점들의 합**이 지배한다.
`κ_nn = 1/d_n² + 1/d_np1²` 공식은 이 취소 효과를 반영하지 않아 항상 κ_nn >> κ_total.

**흥미로운 관측**:
중간점 κ_total은 실제로 "최근접 쌍을 제외한 나머지 영점들의 누적 기여"를 측정함.
이것이 비국소 정보 측정의 다른 방식일 수 있음 (수학자 판단 필요).

**가능한 재설계 방향** (수학자 결정 필요):
1. **측정점 변경**: 중간점 대신 영점에서 δ-오프셋 (기존 Lehmer 방식). 단, 최근접 항 우세한 점에서 측정.
2. **κ_nn 공식 수정**: `κ_nn_올바른 = |−i/(m−γₙ) + i/(m−γₙ₊₁)|²` → 중간점에서 ≈0이라 의미 없음.
3. **관점 전환**: 중간점 κ = "비국소 기여"만을 직접 측정하는 점으로 재해석. ρ(κ_midpoint, gap_next)를 그 자체로 실험 (κ_nn 빼기 불필요).
   → ρ(κ_total, gap) = -0.073 (거의 0), ρ(κ_total, gap_next) = ? 를 측정하면 됨.
